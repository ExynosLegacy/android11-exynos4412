# 1. Purpose

This document defines the documentation standards used throughout the ExynosLegacy Platform project.

Its goal is to ensure that every document follows the same structure, terminology, and quality standards.

---

# 2. Documentation Principles

The project follows these principles:

- Documentation First
- Architecture before Implementation
- Every important decision is documented
- Every document has an owner
- Every document has a version
- Every document can be reviewed
- Documentation evolves together with the code

---

# 3. Document Identification

Every document receives a unique identifier.

Examples:

- DOC-0001
- ADR-0001
- ENG-0001
- RES-0001
- TST-0001
- REL-0001

Identifiers are never reused.

# 5. Versioning

Documentation uses Semantic Versioning.

Examples:

- 1.0.0
- 1.1.0
- 1.2.0
- 2.0.0

---

# 6. Language

The official documentation language is English.

Developer discussions may use Polish.

---

# 7. Markdown Standard

All documentation shall use Markdown.

Headings:

- H1 for document title
- H2 for chapters
- H3 for subchapters

Tables shall be preferred over screenshots whenever possible.

---

# 8. References

Every document should reference related documents whenever applicable.
